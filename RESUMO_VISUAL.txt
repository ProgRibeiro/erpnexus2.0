 3 ENDPOINTS DE API READY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  GET /api/v1/ordens/agenda/
  ├─ Params: data_inicio, data_fim, tecnico, tipo_servico
  ├─ Retorna: Lista agrupada por data → tecnicos → ordens
  └─ Uso: Visão mensal/semanal no admin

  GET /api/v1/ordens/agenda/hoje/
  ├─ Params: nenhum (filtro automático por usuário)
  ├─ Retorna: Array de OS do dia, ordenado por hora
  └─ Uso: Visão de hoje e mobile para técnico

  PATCH /api/v1/ordens/{id}/reagendar/
  ├─ Body: { data_agendada, hora_inicio?, tecnico_responsavel? }
  ├─ Retorna: OS atualizada + cria log automático
  └─ Uso: Reagendar com histórico de alterações
