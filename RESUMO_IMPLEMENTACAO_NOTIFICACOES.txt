## FLUXO DE FUNCIONAMENTO

### 1. Disparar por Evento
```
OrdemServico.save()
  → Signal post_save
    → enviar_email_os_atribuida.delay(os_id)
      → Celery enfileira task
        → Worker executa
          → EmailNotificacao envia email
            → LogNotificacao registra resultado
```

### 2. Agendadas (Celery Beat)
```
Beat scheduler a cada período
  → Dispara tarefa
    → Worker executa
      → Verifica condições (atraso, estoque, etc)
        → Envia notificações
          → Registra logs
```

### 3. API Admin
```
POST /api/v1/notificacoes/logs/testar_notificacao/
  → Valida dados
    → Cria EmailNotificacao
      → Envia imediatamente
        → Registra log
          → Retorna sucesso/erro
```

## NOTIFICAÇÕES IMPLEMENTADAS (7 TIPOS)

1. **OS Atribuída** (instantânea ao técnico)
   - Quando: OS recebe técnico responsável
   - Para: Email do técnico

2. **OS Aprovada** (instantânea ao gerente)
   - Quando: Status muda para "aprovada"
   - Para: Email administrativo

3. **Lembrança Agendamento** (18h diariamente)
   - Quando: 1 dia antes da data agendada
   - Para: Email do técnico

4. **OS Finalizada** (instantânea ao admin)
   - Quando: Status muda para "concluida"
   - Para: Email administrativo

5. **Pagamentos Atrasados** (09h diariamente)
   - Quando: Há pagamentos em atraso
   - Para: Email do financeiro
   - Frequência: Diária

6. **Estoque Baixo** (segunda 08h)
   - Quando: Produto < estoque_minimo
   - Para: Email do estoque
   - Frequência: Semanal

7. **Relatório Pronto** (instantânea ao cliente)
   - Quando: Relatório é gerado
   - Para: Email do cliente

## PRÓXIMOS PASSOS (SETUP)

1. Editar .env com configurações SMTP
2. python manage.py makemigrations notificacoes
3. python manage.py migrate
4. python init_notificacoes.py
5. Iniciar Celery Beat e Worker
6. Testar via API ou admin
