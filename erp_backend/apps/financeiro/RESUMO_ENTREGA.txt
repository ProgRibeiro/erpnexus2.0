✅ REQUISITOS ATENDIDOS (Todos os 5 pontos do brief)

1. ✅ apps/financeiro/signals.py COM TODOS OS SIGNALS

   Signal 1: Quando OS muda para "faturado" → cria Lancamento receita automático
   ├─ @receiver(post_save, OrdemServico)
   ├─ Detecta mudança de status para FATURADA
   ├─ Cria Lancamento tipo RECEITA
   ├─ Conta: Caixa principal
   ├─ Categoria: Receita de servicos
   └─ Status: PENDENTE

   Signal 2: Quando Lancamento status muda para "pago" → atualiza OS
   ├─ @receiver(post_save, Lancamento)
   ├─ atualizar_pagamento_os() - Atualiza status_pagamento
   ├─ atualizar_data_recebimento_os() - Atualiza data_recebimento
   └─ Logging: Detalhado

   Signal 3: Quando OS.data_vencimento passa e não pago → muda para "atrasado"
   ├─ Celery Beat: atualizar_lancamentos_vencidos()
   ├─ Diário (00:00)
   ├─ Encontra Lancamento PENDENTE com data_vencimento < hoje
   └─ Marca como ATRASADO

   Signal 4: Quando DespesaOS é lançada → cria Lancamento despesa automático
   ├─ @receiver(post_save, DespesaOS)
   ├─ criar_lancamento_despesa_os()
   ├─ Tipo: DESPESA
   ├─ Status: PAGO (já realizada)
   └─ Recalcula saldo imediatamente

   Signal 5: Quando OS é cancelada → cancela Lancamento vinculado
   ├─ @receiver(post_save, OrdemServico)
   ├─ sincronizar_lancamento_os()
   ├─ Encontra Lancamentos da OS
   ├─ Marca como CANCELADO (exceto já pagos)
   └─ Atualiza OS.status_pagamento = CANCELADO

   Signal 6: Recalcula saldo da conta quando lançamento é pago
   ├─ @receiver(post_save, Lancamento)
   ├─ recalcular_saldo_conta_ao_pagar()
   ├─ Chama função helper recalcular_saldo_conta()
   ├─ Fórmula: Inicial + Receitas - Despesas
   └─ Previne inconsistências

2. ✅ apps/financeiro/tasks.py - Tarefas Celery

   Task 1: atualizar_lancamentos_vencidos()
   ├─ Agendamento: Celery Beat diário
   ├─ Varrer lançamentos vencidos
   ├─ Marcar como ATRASADO
   ├─ Atualizar OS.status_pagamento = VENCIDO
   └─ Retorna: {lancamentos_atualizados, ordens_atualizadas}

   Task 2: notificar_financeiro_atrasos()
   ├─ Agendamento: Celery Beat diário
   ├─ Notificação para financeiro de atrasos
   ├─ Cria LogNotificacao PAGAMENTO_ATRASADO
   ├─ Incluir dias de atraso e valor
   └─ Retorna: {notificacoes_criadas, valor_total_atraso}

   Task 3: recalcular_saldo_todas_contas()
   ├─ Agendamento: Celery Beat diário
   ├─ Recalcular saldo de todas as contas
   ├─ Para cada ContaBancaria ativa
   └─ Retorna: {total_contas, contas: [...]}

   Helper (não é task): recalcular_saldo_conta(conta_id)
   ├─ Função sincronizada
   ├─ Calcula: Inicial + Receitas_pagas - Despesas_pagas
   ├─ Nunca confie no saldo_atual sem recalcular
   └─ Usada por signals e tasks

3. ✅ Função recalcular_saldo_conta(conta) que:
   ├─ Suma todos os lançamentos pagos (receitas - despesas)
   ├─ Nunca confia no saldo_atual sem recalcular
   ├─ Implementada em tasks.py como função helper
   └─ Chamada automaticamente quando necessário

4. ✅ Registre signals em apps/financeiro/apps.py ready()
   ├─ ✅ Já existia, mantido e verificado
   ├─ import apps.financeiro.signals
   └─ Signals são carregados automaticamente ao iniciar app

5. ✅ Adicione testes em apps/financeiro/tests.py para cada signal
   ├─ 6 testes para signals

   ├─ Cobertura completa de fluxos
   └─ Testes validam comportamento esperado
