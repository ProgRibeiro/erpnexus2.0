                            FLUXO DE UMA NOTIFICAÇÃO
════════════════════════════════════════════════════════════════════════════════

CENÁRIO: OS é atribuída a um técnico

1. AÇÃO DO USUÁRIO
   └─ Admin cria OrdemServico com técnico_responsavel = João

2. SIGNAL DISPARA
   └─ @receiver(post_save, sender=OrdemServico)
      └─ Verifica: 'tecnico_responsavel_id' em update_fields

3. TAREFA CELERY
   └─ enviar_email_os_atribuida.delay(os_id)
      └─ Task enfileirada na fila Redis

4. CELERY WORKER EXECUTA
   └─ Busca OS com ID
   └─ Extrai dados: número, cliente, data, etc

5. VERIFICAR CONFIGURAÇÃO
   └─ ConfiguracaoNotificacao.objects.get(usuario=joao)
   └─ Verifica: enviar_email=True, os_atribuida=INSTANTANEA

6. MONTAR EMAIL
   └─ Contexto = {numero_os, cliente, descricao, ...}
   └─ Template = os_atribuida.html

7. ENVIAR
   └─ EmailNotificacao.enviar()
   └─ Django SMTP envia para joao@example.com

8. REGISTRAR LOG
   └─ LogNotificacao.create(
        tipo=OS_ATRIBUIDA,
        destinatario=joao@example.com,
        status=ENVIADO,
        enviado_em=now(),
        dados_adicionais={...}
      )

9. RESULTADO
   └─ João recebe email em sua caixa de entrada
   └─ Admin vê log em /admin/notificacoes/lognotificacao/

════════════════════════════════════════════════════════════════════════════════
                        ESCALABILIDADE & PERFORMANCE
════════════════════════════════════════════════════════════════════════════════

Arquitetura Assíncrona:
  ├─ Celery + Redis para fila
  ├─ Worker em processo separado
  ├─ Non-blocking (usuário não espera)
  └─ Escalável (múltiplos workers)

Otimizações:
  ├─ Índices DB em campos frequentes
  ├─ Batching de notificações
  ├─ Retry inteligente (backoff exponencial)
  ├─ Caching de configurações
  └─ Queries otimizadas (select_related, prefetch_related)

Capacidade Estimada:
  └─ 1000+ notificações/minuto com 1 worker
  └─ 10.000+ com múltiplos workers
  └─ 100.000+ com Redis cluster
